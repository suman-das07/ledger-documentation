I.

Overview

Stock Guru is a lightweight, front-end inventory management dashboard built entirely with vanilla HTML, CSS, and JavaScript. It allows a shop owner, small business, or individual to record products, track stock levels, monitor inventory value, and manage day-to-day stock operations directly from the browser — with no server, no database, and no external dependency required to run it.

The name reflects its purpose: managing stock in the sense of inventory — quantities, categories, and pricing of physical goods — and is unrelated to financial markets or stock trading. This is stated plainly here so there is no confusion for a reader encountering the name for the first time.

It was built to demonstrate that a genuinely useful, well-considered tool can be created using only the fundamentals of the web platform: the DOM, the Web Storage API, and plain JavaScript logic — without reaching for a framework, a build tool, or a backend.

II.

Why This Project Exists

Most portfolio projects reach for a framework before the fundamentals are proven. Stock Guru intentionally does the opposite. It was built as a demonstration of core front-end engineering competence:

  • Direct DOM controlNo virtual DOM, no templating engine — every update is written by hand.
  • Single source of truthLocal storage is read fresh across two separate HTML pages, never trusted from memory alone.
  • Derived stateDashboard totals and stock status are computed on demand, never cached and left to drift.
  • Original identityA deliberate visual language in place of a default component library look.

For a recruiter or hiring manager, the value here is not that it reinvents inventory software. It is that every interaction — the count rising on the dashboard, the badge changing colour, the table re-sorting — is wired by hand, which makes the underlying logic fully inspectable in the source rather than hidden inside a dependency.

III.

Core Features

  • Live dashboardTotal products, low-stock count, out-of-stock count, and total inventory value, recalculated on every change.
  • Add product formA dedicated page with field validation that rejects incomplete submissions.
  • Persistent storageInventory survives page refreshes and browser restarts on the same device.
  • Status badgesEvery product is labelled In Stock, Low Stock, or Out of Stock automatically.
  • SearchInstant, case-insensitive lookup of a product by name.
  • Category filterNarrow the table to a single category such as Electronics or Networking.
  • Multi-criteria sortBy name, price, quantity, or insertion order, without mutating the underlying dataset.
  • Delete productRemoves an item with an immediate refresh of table and dashboard metrics.
  • Currency formattingIndian numbering conventions with compact notation for large totals.
  • Responsive layoutA dedicated mobile breakpoint reflows the interface into a single column.
IV.

Engineering Highlights and Unique Concepts

Anyone can build a form that writes to local storage. The decisions below are what separate this implementation from a tutorial copy.

  • Re-read on every operationEvery function — renderTable, updateDashboardCards, searchItem, filterCategory, sortProducts, deleteProduct — independently re-reads from storage before acting, so the UI can never show data that is stale relative to what is actually persisted.
  • Non-destructive sortingSorting operates on a shallow copy of the array, so the original insertion order and IDs are always preserved.
  • Self-incrementing identityProduct IDs are derived from the last stored item at load time, keeping identity stable across sessions without a database.
  • Threshold-driven statusStock status is computed at render time from a single quantity value, so the business rule lives in exactly one place.
  • Typographic hierarchyA display face for headings, a barcode-styled face for the tagline, and a monospaced face for tabular data — each chosen with intent.
  • Bilingual identityAn English product name paired with a Hindi tagline, a deliberate localization choice.
  • CSS-only motionThe animated background grid uses a single keyframe rule, with no JavaScript animation loop.
  • Zero build stepThe application runs by opening an HTML file — no install, no bundler, no transpiler.

VIII.

Getting Started

No installation, package manager, or build process is required.

  • 1.Clone the repository — git clone https://github.com/suman-das07/stock-guru.git
  • 2.Open index.html directly in a web browser.
  • 3.Use "Add Item" to record a product, then return to the dashboard to see it reflected instantly.

X.

Known Limitations and Honest Scope

This section exists so that anyone evaluating the project, technically or otherwise, understands exactly what it is and is not.

Data is stored per browser, on a single device — it is not synced across devices or shared between users. There is no authentication layer or access control. There is no backend API or database; all logic runs client-side. Clearing browser storage will permanently remove all recorded inventory data. The project is intentionally scoped as a front-end demonstration, not a production inventory system for a live business.
